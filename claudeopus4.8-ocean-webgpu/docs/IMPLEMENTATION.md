# 实现原理

## 核心思路：把海面运算搬到 GPU

很多 "WebGPU 海面" 示例其实只是把渲染器换成了 `WebGPURenderer`，
波形仍然在 **CPU** 上用 JavaScript 逐顶点循环计算：

```js
// ❌ 常见做法：CPU 逐顶点，主线程每帧遍历几十万顶点
for (let i = 0; i < positions.length; i += 3) {
  positions[i + 1] = sumOfWaves(x, z, t);   // 主线程算
}
geometry.computeVertexNormals();             // 主线程再算一遍法线
positions.needsUpdate = true;                // 每帧重传整个顶点缓冲
```

这种写法有三个瓶颈：① 波形计算占满主线程；② 法线靠 `computeVertexNormals`
全网格重算；③ 每帧把几 MB 顶点数据从 CPU 重新上传到 GPU。顶点数一高就卡。

本项目改为 **TSL（Three Shading Language）节点**，把这套运算写成节点图，
由 Three.js 编译成 **WGSL**，全部在 **GPU** 上执行。CPU 每帧只更新一个时间
`uniform`，顶点缓冲一次上传、永不变动：

```js
// ✅ 本项目：位移写进 positionNode，GPU 顶点着色器并行算
material.positionNode = vec3(
  positionLocal.x.add(disp.x),
  disp.y,
  positionLocal.z.add(disp.z)
);
```

## 1. 程序化随机波谱（顶点着色器）

海面由 **8 个 Gerstner 波**叠加而成。它们不是手工固定的，而是由 `makeWaveBank(seed)`
按**谱**随机生成——这是让海面看起来「随机、不整齐」的关键：

- **随机相位** `φ₀`：每个波在不同位置起峰。若所有波都在原点同时达到峰值，
  叠加后会出现规则的干涉条纹（「很整齐」）；随机相位彻底打破它。
- **方向散布**：各波方向围绕一个主浪向随机散开（三次方权重向主浪向集中，
  少量斜浪），而非几个固定方向。
- **谱分布**：波长按几何分布（少量长波 + 大量短波），振幅按 `len^0.62` 谱衰减，
  长波高、短波细。
- **深水色散** ω = √(g·k)：短波移动更快，叠加更自然。

用**确定性伪随机**（mulberry32）：同一种子 → 同一片海，可复现。点击
**🎲 重新生成**即换一个种子、重建材质节点，得到一片全新随机海面。

Gerstner 波不仅垂直位移，还沿传播方向**横向收拢**，形成尖峰平谷（陡度 `uChop` 控制）：

```
P.x += Σ  Qᵢ·Aᵢ·Dᵢ.x·cos(φᵢ)
P.z += Σ  Qᵢ·Aᵢ·Dᵢ.y·cos(φᵢ)
P.y  = Σ      Aᵢ·sin(φᵢ)
φᵢ = k·(Dᵢ·xz) + ωᵢ·t + φ₀ᵢ          ← φ₀ᵢ 为随机相位
```

横向收拢系数 `Qᵢ` 做了归一化（`1/(kᵢ·N)`），防止 Gerstner 波在高陡度下自交。
波组常量在 JS 里展开进节点图，全局倍率（浪高 / 风速 / 陡度）是运行期可调的 `uniform`。

## 2. 解析 Gerstner 法线（片元着色器）

没有用 `computeVertexNormals`，而是在**片元着色器里**用 Gerstner 法线公式
（GPU Gems）**解析**求法线——与位移在同一次循环里算出，无需多次采样：

```
N = normalize( vec3( −Σ Dx·k·A·cos φ,  1 − Σ Q·k·A·sin φ,  −Σ Dz·k·A·cos φ ) )
```

因为是**逐像素**求值且包含全部谱波（含最短的波），水面高光与反射细节比逐顶点
插值的法线锐利得多——即便网格三角形较大，也能看到细密波纹。

## 3. 海水着色（片元着色器）

最终颜色由若干项合成：

- **水体色**：按高度在深水蓝（`#01080f` 附近）与浅水青之间过渡。
- **菲涅尔反射**：Schlick 近似 `F = (1−N·V)⁵`，掠射角反射更强。
- **天空 / 太阳反照**：沿反射方向 `reflect(-V, N)` 采样**与天空穹顶共用**的
  `skyColorFn`。因为天空里画了太阳本体与大气辉光，反射回来便是物理一致的
  **太阳波光**——低日照角度时在水面拉出长长的闪烁光带。
- **次表面散射**：迎着太阳的浪尖叠加青绿，模拟阳光穿透薄浪。
- **湍流海沫**：先由浪尖高度 + 陡坡判断「哪里可能起沫」，再乘以一层**随时间流动的
  噪声场**（`mx_noise_float`）过阈值，把泡沫切成**不规则斑块**而非沿波峰成排；
  叠加高频细沫噪声做出颗粒感。这样泡沫看起来是自然散布的，没有人工排布痕迹。
- **距离雾**：远处按距离平滑融入地平线天空色。

天空穹顶与海面反射复用同一个 `skyColorFn(dir)`，保证「水里的太阳」和
「天上的太阳」永远一致——这是太阳波光看起来真实的关键。

## 4. 暂停与时间

内置的 `time` 节点无法暂停，因此用一个自管理的 `uTime` uniform：
动画循环里 `uTime.value += delta`，暂停时停止累加。波形与噪声都引用 `uTime`，
于是暂停时画面凝固、参数仍可调。

## 5. 降级处理

入口先用 `WebGPU.isAvailable()` 检测；不支持时显示友好提示卡片而非白屏。
`startOcean()` 整体包在 `try/catch` 里，初始化或着色器编译异常也会被捕获并提示。

## 性能

| 项 | 值 |
| --- | --- |
| 网格 | 900×900，320×320 分段（≈ 103k 顶点） |
| 波数 | 8 个 Gerstner 波（随机谱），逐像素解析法线一次求值 |
| 波形 / 法线 / 着色 | 全部 GPU（TSL → WGSL） |
| CPU 每帧开销 | 仅更新 1 个时间 uniform + OrbitControls |

> **波数 vs 帧率**：片元侧逐像素求解 Gerstner 法线，开销随波数线性增长。
> 8 波在主流独显上可轻松跑满 60 FPS；若在集显/共享 GPU 上偏卡，可调小
> `main.js` 顶部的 `N_WAVES`（如 6），或降低画布像素比。
