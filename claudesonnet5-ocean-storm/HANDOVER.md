# 任务交接文档 (HANDOVER.md) — claudesonnet5-ocean-storm

## 1. 项目基本状态

本项目根据 `RULES.md` 规范创建，是一个基于 **Three.js (WebGL, 自定义 GLSL ShaderMaterial)** 实现的海洋 + 天气 + 风暴模拟场景。

- **开发负责人**：Claude Sonnet 5 (AI Coding Assistant)
- **端口**：3015（`npm run dev:sonnet-storm` 或根目录 `npm run dev`）
- **技术核心与差异化定位**：与目录下其他海洋项目（`antigravity-ocean-storm`、`gpt5.5-ocean-weather`、`gpt5.5-ocean-webgpu` 等）相比，本项目的天气不是"全局预设切换"或"滑块连续插值"，而是**风暴单元（Storm Cell）在世界坐标系中独立生成、漂移、消散**，天气效果（风速/浪高/降雨/云层/雷电）完全由相机到最近风暴单元的**空间距离**实时推导。玩家可以看到风暴从地平线移近、途经、远离的完整时空过程，而不是场景整体瞬间"变天"。

## 2. 详细技术方案

### A. 移动风暴单元系统 (`src/weather/StormSystem.js`)

- 每个风暴单元拥有独立的世界坐标 `(x, z)`、核心半径、衰减半径、生命周期（生长 20s → 成熟 30~70s，强度带轻微正弦抖动 → 衰减 24s 后销毁），以及基于 fbm 噪声扰动的漂移航向（非直线，路径会自然弯曲）。
- 最多同时存在 2 个风暴单元；自动模式下按随机冷却时间在相机外围环形区域生成，生成时会朝相机方向偏移一个随机角度，因此大多数风暴会"擦肩而过"而非每次正面命中。
- 每帧计算相机到所有风暴单元的距离，取最大 `影响值 = 强度 × falloff(距离, 核心半径, 衰减半径)`，由此推导出统一的天气快照 `state`：风向/风速、降雨强度、云量、天空变暗程度、雾浓度、曝光、雷击概率、最近风暴距离与方位。
- 全局"基础气候"（弱风、少量浮云）始终存在，通过 fbm 噪声缓慢漂移，保证无风暴时海面也不会死寂。
- 支持 GUI 手动接管（滑块强制指定风暴强度，跳过单元生长动画，便于测试/展示）与"就近生成风暴"/"清空天空"按钮。

### B. 海洋渲染 (`src/ocean/Ocean.js`)

- 双层 Gerstner 波：4 条固定方向的"涌浪"（swell，低幅、始终存在）+ 5 条随实时风向旋转、随风速增幅的"风浪"（chop）。法线通过解析偏导数（切线/副切线叉乘）求出，Jacobian 行列式驱动泡沫遮罩。
- 网格通过 `followCamera()` 每帧吸附到相机 XZ（按格子尺寸取整），配合顶点着色器中用 `modelMatrix` 平移量还原世界坐标作为波相位输入，实现"无限海面"且波形不随网格吸附而跳变。
- 降雨会在片元着色器中叠加高频 fbm 扰动法线（雨滴打在水面的视觉暗示），风暴闪电命中点会给附近海面叠加短暂的高光衰减。

### C. 天空穹顶 (`src/sky/SkyDome.js`)

- 跟随相机的大球体，两octave fbm 云层随风向滚动。最多 3 个风暴单元会被投影为"按真实方位角变暗/增稠"的天空色块（`uCellDir`/`uCellAngular`/`uCellStrength`），也就是说风暴逼近方向的天空会先于地面天气变化而变暗，是本项目里天空与地面风暴系统联动的关键细节。

### D. 降雨 (`src/weather/RainField.js`)

- `InstancedMesh` 常驻 5200 条雨丝，顶点着色器仅依据 `uCameraPos` + 每实例哈希值计算下落/风偏/重生，逐帧零 CPU 开销；密度变化通过哈希阈值裁剪而非改变实例数，帧时间稳定。

### E. 闪电 (`src/weather/LightningSystem.js`)

- 递归中点位移生成主干 + 分叉，几何体本身颜色值 >1（HDR），完全依赖 `UnrealBloomPass` 产生辉光，无需手工做多层描边模拟发光。闪电触发概率、坐标均来自 `StormSystem.rollLightningStrike()`，命中点定位在最强风暴单元核心半径内。

### F. 程序化音频 (`src/audio/AmbientAudio.js`)

- 全部由 WebAudio 合成噪声得到，无音频素材：风/雨为持续滤波噪声床，增益跟随天气状态；雷声是低通扫频噪声脉冲，按**真实声速**（距离 / 340 m/s）延迟播放，风暴较远时呈现"先见闪光、数秒后才听到雷声"的效果。受浏览器自动播放策略限制，音频需用户点击左上角按钮解锁。

### G. 镜头 (`src/main.js`)

- `OrbitControls` 的 `target` 与 `camera.position` 每帧被施加同一个"巡航漂移量"（航向受 fbm 噪声轻微弯曲），实现"缓慢飞越风暴海域"的运镜效果，同时保留用户可自由环绕张望的交互；相机高度被限制在海面涌浪高度之上，避免穿入水面。

## 3. 已知问题与后续优化

- 泡沫的视觉形态偏"大块状"，可以引入更高频的 fbm 细节或屏幕空间抗锯齿来改善浪尖碎浪的颗粒感。
- 暂无浮体/船只与海面的物理交互（其他姊妹项目有此功能），后续可复用 `Ocean.getSwellHeight()` 扩展浮力采样。
- 降雨目前只有实例化雨丝，未叠加屏幕空间雨幕后处理，近景可考虑再加一层水渍/雨滴后期特效增强沉浸感。
- 闪电分叉与主干共用同一套随机数生成器（`Math.random`），未做可复现的种子控制，如需要"回放同一场风暴"需要改为自带种子的 PRNG。
