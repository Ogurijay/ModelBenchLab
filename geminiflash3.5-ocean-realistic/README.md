# Gemini 3D 写实海洋物理与渲染模拟器

本测试用例展示了 Gemini 开发的基于 **Three.js** 的 3D 写实海面仿真，支持物理浮力（Buoyancy）实时交互与多时段天色插值，旨在评估智能体在高水准物理计算与定制 WebGL Shader 材质上的开发质量。

---

## 核心特性与架构设计

### 1. 完整的 Gerstner 浪方程（Trochoidal Waves）
不同于只在垂直方向做正弦偏移的简单波浪，本项目在 GPU（材质着色器）与 CPU（物理结算）端同步实现了完整的三维 Gerstner 波形。通过控制陡度因子（Choppiness），波峰在横向上朝脊线挤压、收窄变尖，能够表现出更具攻击性和真实感的汹涌浪潮。

### 2. 精确的解析求导法线 (Analytic Normals)
为了避免片元着色器使用有限差分偏导（`dFdx/dFdy`）在远景或摄像机运动时引入大量像素级的闪烁噪点，我们在顶点着色器中对 Gerstner 三维物理位移方程进行了**数学一阶偏导数解析求解**。通过计算切向量（Tangent）与副切向量（Bitangent）的叉乘得到精确的法向量：
* **优势**：水面法线极其平滑，受相机距离影响极小，抗锯齿闪烁能力显著提升，且相比运行时数值差分更具性能优势。

### 3. CPU 端定点迭代（Fixed-point Iteration）逆反解算
由于 Gerstner 波存在横向位移，给出一个世界坐标 $(X_{world}, Z_{world})$，其未发生偏移前的原始坐标 $(X_{orig}, Z_{orig})$ 无法通过简单反函数求得。
在 CPU 端进行物理浮力采样时，我们引入了**定点反向迭代算法**：
* 通过 3 次快速迭代收敛，精准还原出当前物体所在水平面坐标对应的海浪高度与法线。这使物体在水面大范围波动时，依然能够分毫不差地贴紧波浪表面。

### 4. CPU-GPU 双重物理反馈与对齐
每个抛入海中的小球与救生圈通过重力、浮力（根据浸入深度）和水阻力进行物理二阶积分。物体的上方向轴（UP 轴）利用四元数球形线性插值（`slerp`）与海面的波浪解析法向量平滑对齐，还原出完美的漂浮物起伏和左右摇摆。

### 5. 时段联动大气散射（Time of Day System）
包含“正午阳光”、“金色黄昏”、“静谧月夜”、“雷暴海啸”四种预设。切换预设时，太阳位置变化，天空 Shader（日面光晕、大气色度散色）与海水 Shader（菲涅尔天空反射、太阳高光、次表面散射透光颜色）会完全同步无缝插值。

## 项目结构

```text
geminiflash3.5-ocean-realistic/
├── index.html           # 全屏 Canvas 容器与玻璃拟态控制板
├── package.json         # 依赖与打包脚本
├── README.md            # 本说明文档
├── src/
│   ├── main.js          # Three.js 场景管理、物理与渲染主循环、交互投球
│   ├── styles.css       # 磨砂玻璃拟态 UI 布局及响应式控制
│   ├── ocean/
│   │   ├── waves.js     # 多重波参数生成、CPU 高度与法线采样、定点反解迭代器
│   │   └── materials.js # 高清海面 Shader (解析偏导法线、双 specular、SSS) 与天空 Shader
│   └── ui/
│       └── panel.js     # UI 事件响应、滑块监听、状态看板同步
└── tests/
    └── buoyancy.test.js # CPU 仿真稳定性与物理边界单元测试
```

## 运行与验证说明

在当前目录下（`geminiflash3.5-ocean-realistic/`）打开终端，可执行以下命令：

### 1. 运行物理核心单元测试
```bash
npm run test
```
* 验证 Gerstner 参数约束、浮力反解迭代器的收敛性和法线单位归一化。

### 2. 启动本地开发服务（游玩与视觉验证）
```bash
npm run dev
```
* 浏览器打开终端输出的本地地址（默认 `http://127.0.0.1:5173` 或同等端口）。
* 可以在 Canvas 空白处任意点击，实时抛入小球，观察小球在海浪里随波逐流的重力浮力状态。
* 切换天气，观察整体漫反射、次表面透光、菲涅尔反射以及高光颜色的和谐变化。

### 3. 构建前端打包
```bash
npm run build
```
* 检验打包发布流程是否有错。
